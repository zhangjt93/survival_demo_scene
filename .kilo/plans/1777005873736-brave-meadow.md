# Portal 近裁面裁剪问题改进方案

## 问题分析

当前 `_update_portal_camera_near_clip_plane` 的问题：

1. **只用传送门中心点计算距离** — 第49行用 `target_portal.global_position`（即出口传送门的中心点）投影到相机 forward 方向上来计算 near 值
2. **倾斜夹角时出问题** — 当虚拟相机与出口传送门表面有倾斜夹角时，传送门四边形各个角到相机 forward 方向的投影距离不同。用中心点距离无法覆盖整个传送门四边形，导致部分传送门画面被近裁面裁掉

## 推荐方案：四角点投影法（Donitzo 方案）

参考 Donitzo/godot-simple-portal-system 的实现，核心思路：

### 核心算法

不再用传送门中心点计算距离，而是：

1. **获取出口传送门四边形四个角的世界坐标** — 使用传送门 `MeshInstance3D` 的 AABB 四个角（XY 平面，Z=0），转换到世界空间
2. **计算各角到虚拟相机的投影距离** — 每个角的位置向量点乘虚拟相机的 forward 向量（-basis.z），得到沿视线方向的投影距离
3. **取最小值作为 near** — 选择四个投影距离中的最小值（最靠近相机的那个角），减去一个偏移量 `exit_near_subtract`，作为最终的 near 值

### 关键改进

| 当前实现 | 改进后 |
|---------|-------|
| 仅用 `target_portal.global_position`（1个点） | 用传送门四边形的4个角 |
| 只考虑中心点的投影距离 | 考虑所有角的投影距离，取最保守值 |
| 无偏移量控制 | 添加 `exit_near_subtract` 参数允许微调 |

### 具体代码修改

1. **新增导出变量**：
   - `@export var exit_near_subtract: float = 0.05` — 近裁面偏移量，用于微调

2. **新增常量**：
   - `const EXIT_CAMERA_NEAR_MIN: float = 0.01` — 最小近裁面距离

3. **重写 `_update_portal_camera_near_clip_plane` 方法**：

```gdscript
const EXIT_CAMERA_NEAR_MIN: float = 0.01
@export var exit_near_subtract: float = 0.05

func _update_portal_camera_near_clip_plane(camera: Camera3D):
    # 获取出口传送门四边形的四个角（XY平面，Z=0）
    var portal_size = target_portal.portal_visual.size
    var half_w = portal_size.x / 2.0
    var half_h = portal_size.y / 2.0
    
    # 传送门局部坐标系的四个角
    var corners_local = [
        target_portal.global_transform * Vector3(-half_w, -half_h, 0),
        target_portal.global_transform * Vector3( half_w, -half_h, 0),
        target_portal.global_transform * Vector3( half_w,  half_h, 0),
        target_portal.global_transform * Vector3(-half_w,  half_h, 0),
    ]
    
    var cam_forward = -camera.global_transform.basis.z.normalized()
    var cam_pos = camera.global_position
    
    # 计算所有角到相机的投影距离，取最小值
    var min_dist = INF
    for corner in corners_local:
        var dist = (corner - cam_pos).dot(cam_forward)
        if dist < min_dist:
            min_dist = dist
    
    # 设置 near，减去偏移量确保包含整个传送门
    camera.near = max(EXIT_CAMERA_NEAR_MIN, min_dist - exit_near_subtract)
```

4. **清理不再需要的变量**：
   - `near_clip_offset`（第8行）可以保留或移除，如果替换为新机制则不再使用
   - `min_near`（第9行）不再需要，用 `EXIT_CAMERA_NEAR_MIN` 替代

### 为什么这样做能解决问题

- 当虚拟相机与传送门表面成倾斜夹角时，四个角到相机的距离各不相同
- 取最小值确保即使最「近」的那个角也能被完全包含在视锥体内
- 减去 `exit_near_subtract` 提供一个安全边际，防止浮点精度导致的边缘裁切

## 不选用的方案

### 1. 倾斜投影矩阵（Oblique Near Plane）
- Godot 4.x 移除了 `set_override_projection`，且 `Camera3D` 没有原生支持
- 需要修改引擎源码（提案 #1863、#2713），侵入性太强

### 2. 屏幕空间着色器剔除
- 作为补充方案可行，但无法独立解决近裁面问题
- 需要与上述方案配合使用，复杂度增加

## 注意事项

- 本方案假设 `portal_visual`（CSGBox3D）的中心在传送门节点的原点
- 如果传送门有非均匀缩放，需要额外处理（目前代码中 `_thicken_portal_if_necessary_to_prevent_camera_near_cull` 已有类似处理）
- 改进后的 near 值会比原来更保守（可能更小），在极端角度下仍然可能无法完美覆盖 — 但相比当前方案已有本质改善
