## 实现方案

### 一、基础物理传送

#### 检测原理
利用 `front_area`（z 轴负方向）和 `rear_area`（z 轴正方向）追踪物体穿过传送门的过程：
1. 物体进入 `front_area` → 记录为"从前方接近"
2. 物体同时进入 `rear_area` → 判定为"穿过传送门平面" → 执行传送

#### 传送变换
```
物体新变换 = 目标传送门全局变换 × (当前传送门全局变换的逆 × 物体全局变换)
```
这与 VirtualCamera 的位置计算逻辑完全一致（`portal.gd:100-102`）。

#### 速度变换
将物体的线速度和角速度也通过传送门的旋转变换进行映射。

#### 需要修改的文件
- `scripts/portal.gd` — 添加传送逻辑
- `scripts/portal_area.gd` — 添加进入/退出信号处理

### 二、防循环传送（往回走处理）

#### 问题
角色从 Portal A 传送到 Portal B 后，仍在 Portal B 的检测区域内，如果往回走会立刻被传送回 Portal A，形成死循环。

#### 方案：出口冷却标记
- 维护一个字典 `_just_teleported: Dictionary = {}`，key 为物体实例 ID，value 为刚传送到的 Portal 对象
- 传送后，将物体标记为 `just_teleported[object_id] = target_portal`
- 当物体**完全退出**该传送门的前后区域后，清除标记
- 传送前检查：如果物体在该传送门的 `_just_teleported` 中，跳过传送

```
流程：
1. 角色从 PortalA 前方穿过 → 传送到 PortalB
2. 角色被标记 just_teleported[id] = PortalB
3. 角色在 PortalB 区域内往回走 → PortalB 检测到穿越 → 检查标记 → 跳过
4. 角色完全离开 PortalB 区域 → 清除标记
5. 角色再次从 PortalB 前方穿过 → 正常传送
```

这样角色可以在传送后自由选择方向，只要先离开传送门区域就能正常再次传送。

### 三、双传送门可见时的物体镜像（Ghost 渲染）

#### 问题
当一个物体在 Portal A 附近时，如果 Portal A 和 Portal B 都在主摄像机视野内：
- 通过 Portal B 看到物体（VirtualCamera 已经渲染）✅ 已有
- 在 Portal A 旁也能看到物体本体 ✅ 已有
- **但物体"穿过"传送门平面后，露在另一侧的部分需要渲染为"镜像体"**

#### 方案：Ghost 节点

为进入传送门区域的物体创建纯视觉的 Ghost 副本：

1. **Ghost 创建时机**: 物体进入 `front_area` 或 `rear_area` 时
2. **Ghost 位置**: 通过传送门变换计算在目标传送门侧的对应位置
3. **Ghost 内容**: 仅复制 MeshInstance3D 等视觉节点（不复制碰撞体）
4. **Ghost 可见层**: 设置为 `cull_layer`，使其只在传送门 VirtualCamera 中可见，主摄像机不可见
5. **Ghost 更新**: 每帧同步变换
6. **Ghost 销毁**: 物体离开区域或传送后销毁

#### 实现细节

```gdscript
# 在 portal.gd 中维护
var _ghosts: Dictionary = {}  # object_id -> Node3D (ghost root)

func _create_ghost_for(body: Node3D):
    if _ghosts.has(body.get_instance_id()):
        return
    var ghost_root = Node3D.new()
    _clone_visual_nodes(body, ghost_root)
    # ghost 节点设置到 target_portal 的场景中
    target_portal.add_child(ghost_root)
    _ghosts[body.get_instance_id()] = ghost_root

func _update_ghost_transform(body: Node3D):
    var ghost = _ghosts.get(body.get_instance_id())
    if ghost:
        var rel = self.global_transform.affine_inverse() * body.global_transform
        ghost.global_transform = target_portal.global_transform * rel
```

#### Ghost 渲染层策略
- Ghost 节点的 `visible` 属性和 `layer` 设置确保：
  - 在目标传送门的 VirtualCamera 中可见（通过 cull_layer）
  - 在主摄像机中不可见（不在 layer 1）
- 这样物体"探出"传送门的部分通过 Ghost 在另一侧可见
